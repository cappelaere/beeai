# Model Display Feature

## Overview

Each assistant response now displays which AI model generated it at the bottom of the message. This provides transparency about which model was used for each response.

## Implementation

### Backend Changes

**File:** `agent_ui/agent_app/views.py`

Added model name mapping and included it in the chat API response:

```python
# Get readable model name
model_names = {
    "anthropic:claude-sonnet-4-5": "Claude Sonnet 4.5",
    "anthropic:claude-opus-4": "Claude Opus 4",
    "anthropic:claude-3-5-sonnet-20241022": "Claude 3.5 Sonnet",
    "anthropic:claude-3-5-opus-20241022": "Claude 3.5 Opus",
    "anthropic:claude-3-haiku-20240307": "Claude 3 Haiku"
}
model_display_name = model_names.get(model_id, model_id.replace("anthropic:", "").replace("-", " ").title())

return JsonResponse({
    ...
    "model_name": model_display_name,
    ...
})
```

### Frontend Changes

**File:** `agent_ui/static/js/chat.js`

1. Updated `appendMessage()` function to accept `modelName` parameter
2. Added model info display for assistant messages:

```javascript
// Add model name at bottom for assistant messages
if (role === "assistant" && modelName && !isCommand) {
  var modelInfo = document.createElement("div");
  modelInfo.className = "model-info";
  modelInfo.innerHTML = '<svg width="12" height="12" ...></svg> ' + modelName;
  modelInfo.setAttribute("aria-label", "Response generated by " + modelName);
  div.appendChild(modelInfo);
}
```

3. Updated all `appendMessage()` calls to pass `model_name` from API response

**File:** `agent_ui/static/css/theme.css`

Added styling for the model info display:


## Visual Design

The model name appears at the bottom of each assistant message:

```
┌─────────────────────────────────────┐
│ Here is the response...             │
│                                     │
│ [Response content]                  │
│                                     │
│ ⚙️ Claude Sonnet 4.5                │
└─────────────────────────────────────┘
```

**Features:**
- Subtle gray background
- Small text size (0.75rem)
- Left border indicator
- Icon (gear/settings symbol)
- Dark theme support
- Section 508 mode support (larger text: 0.875rem)

## Accessibility

- **ARIA label**: "Response generated by [Model Name]"
- **Screen reader friendly**: Announced after message content
- **Keyboard accessible**: Part of message container
- **High contrast**: Works in both light and dark themes
- **Section 508 mode**: Larger text (0.875rem) and padding

## Behavior

- **Only shown for assistant messages** (not user messages)
- **Not shown for command responses** (e.g., `/help`, `/metrics`)
- **Displayed for all chat modes** (normal, cached responses)
- **Persists in chat history** (though not currently stored in database)

## Model Name Mapping

| Internal ID | Display Name |
|-------------|--------------|
| `anthropic:claude-sonnet-4-5` | Claude Sonnet 4.5 |
| `anthropic:claude-opus-4` | Claude Opus 4 |
| `anthropic:claude-3-5-sonnet-20241022` | Claude 3.5 Sonnet |
| `anthropic:claude-3-5-opus-20241022` | Claude 3.5 Opus |
| `anthropic:claude-3-haiku-20240307` | Claude 3 Haiku |

For unknown models, the system automatically:
1. Removes `anthropic:` prefix
2. Replaces hyphens with spaces
3. Applies title case

Example: `anthropic:new-model-name` → `New Model Name`
