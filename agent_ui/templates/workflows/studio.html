How to start a new workflow

As an admin, open Workflows in the sidebar and click Workflow Studio. Fill the form below; the AI generates a BPMN diagram and workflow code. For more detail, see Docs → Workflow Development.

What Gets Created

When you create a workflow, the following will be generated:

  • Workflow Directory: workflows/<workflow_id>/
  • Metadata: workflows/<workflow_id>/metadata.yaml
  • Implementation: workflows/<workflow_id>/workflow.py (AI-generated)
  • Documentation: workflows/<workflow_id>/README.md (AI-generated)
  • Diagram: workflows/<workflow_id>/currentVersion/workflow.bpmn (or v1/) (BPMN 2.0)

After Creation

Your workflow will:

  • Appear in the Workflow List
  • Be executable via Flo agent
  • Have complete documentation
  • Support human-in-the-loop tasks

Tips

  • Choose a descriptive workflow ID (can't be changed later)
  • Be specific about which agents should handle each step
  • Clearly indicate where human review is needed
  • Define success/failure criteria for each step
  • Include error handling and retry logic
  • Reference Workflow Guidelines for best practices

Example Workflows

  • Bidder Verification: SAM + OFAC checks with human review
  • Property Listing: Validation → Compliance → Approval
  • Document Review: Upload → Analysis → Multi-reviewer approval
